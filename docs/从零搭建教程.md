## 序言：这个智能体能帮你做什么

在你开始写代码之前，先用大白话说清楚：这玩意儿到底能干什么？

用一句话概括：**你告诉它一个话题或者丢给它一篇别人写的好文章，它帮你自动完成「扒内容 → AI 改写 → 合规检查 → 自动配图 → 一键发布到微信公众号」的全流程。**

具体来说，它能做这些事：

| 环节 | 功能 | 用什么技术实现 |
|------|------|---------------|
| 获取素材 | 从微信公众号、知乎、头条等平台扒文章 | Playwright 浏览器自动化 / Scrapling |
| AI 改写 | 把别人的文章用自己的话重写，去 AI 味、口语化 | OpenAI GPT-4 / 国内大模型 |
| 标题优化 | 自动生成多个吸引人的标题供你选 | Prompt 工程 + LLM |
| 封面图 | 自动搜索无版权图片 或 用 DALL-E 生成封面 | Unsplash/Pexels/Pixabay + DALL-E |
| 合规检查 | 检查敏感词、广告法违禁词、AI 深度审核 | 正则 + LLM 双层检查 |
| 排版转换 | Markdown 自动转成微信公众号富文本格式 | 自定义格式化器 |
| 发布文章 | 自动上传素材、创建草稿、推送到公众号 | 微信公众号官方 API |

**整个流程示意图：**

```
[选择话题/源文章] → [扒取原文内容] → [AI改写去味] → [合规检查] → [自动配图] → [自动排版] → [一键发布到公众号]
```

## 第零章：开工前的准备——你需要什么东西

### 硬件要求

- 一台能上网的电脑（Windows/Mac/Linux 都行）
- 至少 8GB 内存（浏览器自动化比较吃内存，16GB 更佳）
- 硬盘剩余空间 >= 10GB

### 软件账号准备

在开始写代码之前，你需要先搞到下面这些东西。**新手最容易卡在这一步，请认真看。**

#### ① 必选：OpenAI API Key（每月花几十块）

这是 AI 的大脑，用来生成文章和画图。

**怎么搞到：**
1. 打开 [platform.openai.com](https://platform.openai.com)
2. 注册一个 OpenAI 账号（需要科学上网）
3. 进入 API Keys 页面，点击「Create new secret key」
4. 复制保存这个 Key（格式像 `sk-proj-xxxxx`，只显示一次）
5. 充值：在 Billing 页面绑定信用卡，充值 5~10 美元就够了

> **新手注意**：OpenAI 的 API 需要科学上网才能访问。如果你没有科学上网条件，可以考虑使用国内的 API 代理服务（比如 API2D、OhMyGPT 等），它们的用法完全一样，只是把地址换成国内能访问的 URL。

> **费用参考**：用 GPT-4o 生成一篇 2000 字的文章，大约花费 0.1~0.3 元人民币。DALL-E 画一张封面图大约 0.3 元。整体成本很低。

#### ② 必选：微信公众号 AppID 和 AppSecret

这是发布文章的钥匙，用来调用微信的官方接口。

**怎么搞到：**
1. 首先你当然要有一个微信公众号（订阅号或服务号都行）
2. 登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)
3. 左侧菜单 → 设置与开发 → 基本配置
4. 你会看到「AppID」和「AppSecret」两个值
5. AppID 直接可见，AppSecret 需要点击「重置」获取（会发到你绑定的微信上确认）
6. **重要**：把这两个值记下来！AppSecret 只显示一次！

> **新手注意**：
> - 个人订阅号的 API 权限有限，但「发布文章」这个能力是有的，足够用了
> - 企业订阅号和服务号的权限更完整
> - 如果还没有公众号，先去注册一个：[mp.weixin.qq.com](https://mp.weixin.qq.com)

#### ③ 推荐：无版权图片 API（免费，不花钱）

用来给文章自动配封面图，不用自己到处找图。

**方案一（最简单）：Pexels API**
1. 打开 [pexels.com/api](https://www.pexels.com/api)
2. 注册账号，申请 API Key
3. 每月有 20000 次免费请求，完全够用

**方案二：Pixabay API**
1. 打开 [pixabay.com/api/docs](https://pixabay.com/api/docs)
2. 注册账号获取 Key
3. 完全免费，无请求次数限制

**方案三：Unsplash API**
1. 打开 [unsplash.com/developers](https://unsplash.com/developers)
2. 注册开发者账号，创建应用
3. 每小时 50 次免费请求

> **建议**：至少搞到 Pexels 的 Key，它最稳定、图也好看。三个都搞到更好，系统会自动从多个来源聚合搜图。

#### ④ 你的电脑需要安装的软件

- **Python 3.10 或更新版本**
  - 下载地址：[python.org](https://www.python.org/downloads/)
  - **安装时一定要勾选「Add Python to PATH」！！！** 不勾的话后面全是坑
  - 安装完打开终端（Win + R → 输入 `powershell` → 回车），输入 `python --version`，能看到版本号就说明装好了

- **Node.js 18 或更新版本**（用于 Playwright 浏览器自动化）
  - 下载地址：[nodejs.org](https://nodejs.org)
  - 选择 LTS 版本即可
  - 安装完终端输入 `node --version` 验证

- **Git**（用于下载代码和版本管理）
  - 下载地址：[git-scm.com](https://git-scm.com)
  - 一路下一步默认安装即可
  - 终端输入 `git --version` 验证

- **一个趁手的代码编辑器**
  - 推荐 VS Code：[code.visualstudio.com](https://code.visualstudio.com/)
  - 免费、好用、中文界面

### 1.2 创建 Python 虚拟环境

**什么是虚拟环境？** 简单说就是给每个项目搞一个独立的 Python 小房间，在这个房间里安装的包不会影响其他项目，也不会把你的系统 Python 搞乱。

```powershell
# 创建虚拟环境（这个命令会在项目里生成一个 venv 文件夹）

```

执行后不会有任何输出，但文件夹里会多出一个 `venv` 目录，这就对了。

**激活虚拟环境：**

```powershell
# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```

激活成功后，终端最前面会出现 `(venv)` 字样，说明你已经进入虚拟环境了。

> **新手常见坑**：
> - 如果 PowerShell 报错「无法加载文件...因为在此系统上禁止运行脚本」，先执行 `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`，输入 Y 确认，然后再激活。
> - 每次打开新终端都要重新激活虚拟环境！这是个好习惯。

## 第三章：把 AI 大脑接进来

> 这一章是整个系统最核心的部分——让 AI 帮你写文章。我们先从最简单的 LLM 调用开始。

### 3.1 理解大模型 API 调用的原理

在写代码之前，先理解两个关键概念：

**什么是 API 调用？** 你不是打开 ChatGPT 网页聊天，而是通过代码发请求给 OpenAI 的服务器。你的请求包含「系统指令 + 用户问题」，服务器返回 AI 的回答。

**什么是 Prompt？** 就是你发给 AI 的指令。同样的 AI 模型，Prompt 写得好不好，结果天差地别。比如：

- 烂 Prompt：「写一篇关于 AI 的文章」
- 好 Prompt：「你是一位资深科技编辑，请以『AI 如何改变普通人生活』为主题，写一篇 2000 字的公众号文章。要求：口语化、有具体案例、分 3 个小标题、每个小标题下不少于 3 段。」

**调用一次的成本大概是多少？**
- GPT-4o：输入约 0.018 元/千字，输出约 0.07 元/千字
- 写一篇 2000 字的文章，大约花费 0.1~0.3 元
- 每天写一篇，一个月也就 3~10 块钱

### 3.2 编写 AI 服务模块

创建 `src/services/ai_service.py`：

```python
"""
AI 服务模块 - 统一的大模型调用接口

支持 OpenAI / Azure / 国内代理 等多种 API
提供自动重试、速率限制、Token 统计等能力
"""

import os
import time
import json
import logging
from typing import Optional

import requests

logger = logging.getLogger(__name__)


class AIService:
    """AI 服务统一调用入口"""

    def __init__(
        self,
        api_key: str = "",
        base_url: str = "",
        model: str = "gpt-4o",
        max_retries: int = 3,
        timeout: int = 60,
    ):
        # 优先使用传入参数，否则从环境变量读取
        self.api_key = api_key or os.getenv("AI_API_KEY", "")
        self.base_url = base_url or os.getenv("AI_API_BASE", "https://api.openai.com/v1")
        self.model = model or os.getenv("AI_MODEL", "gpt-4o")
        self.max_retries = max_retries
        self.timeout = timeout

        if not self.api_key:
            raise ValueError("AI_API_KEY 未配置！请在 .env 文件中设置 AI_API_KEY")

    def chat(
        self,
        messages: list[dict],
        temperature: float = 0.7,
        max_tokens: int = 4000,
    ) -> str:
        """
        发送聊天请求并返回回复文本

        参数:
            messages: 消息列表，格式 [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}]
            temperature: 随机性，0=保守 1=创意（建议写文章用0.7）
            max_tokens: 最大输出长度

        返回:
            AI 的回复文本
        """
        url = f"{self.base_url}/chat/completions"
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json",
        }
        payload = {
            "model": self.model,
            "messages": messages,
            "temperature": temperature,
            "max_tokens": max_tokens,
        }

        last_error = ""
        for attempt in range(self.max_retries):
            try:
                logger.info(f"AI 请求: model={self.model}, 第{attempt+1}次尝试")
                resp = requests.post(
                    url, headers=headers, json=payload, timeout=self.timeout
                )

                if resp.status_code == 200:
                    data = resp.json()
                    content = data["choices"][0]["message"]["content"]
                    # 打印 Token 用量
                    usage = data.get("usage", {})
                    logger.info(
                        f"AI 响应成功: 输入={usage.get('prompt_tokens')} tokens, "
                        f"输出={usage.get('completion_tokens')} tokens"
                    )
                    return content

                elif resp.status_code == 429:
                    # 速率限制，等待后重试
                    wait_time = 2 ** attempt
                    logger.warning(f"速率限制(429)，{wait_time}秒后重试...")
                    time.sleep(wait_time)
                    continue

                else:
                    last_error = f"HTTP {resp.status_code}: {resp.text[:200]}"
                    logger.error(last_error)

            except requests.exceptions.Timeout:
                last_error = f"请求超时 (>{self.timeout}秒)"
                logger.warning(last_error)
            except Exception as e:
                last_error = str(e)
                logger.error(f"AI 请求异常: {e}")

            if attempt < self.max_retries - 1:
                time.sleep(1)

        raise Exception(f"AI 调用失败（已重试{self.max_retries}次）: {last_error}")

    def generate_article(
        self, topic: str, style: str = "professional", word_count: int = 2000
    ) -> dict:
        """
        根据主题生成一篇公众号文章

        参数:
            topic: 文章主题/关键词
            style: 写作风格 (professional/casual/news/storytelling/tutorial)
            word_count: 目标字数

        返回:
            {"title": "标题", "content": "正文", "summary": "摘要"}
        """
        system_prompt = """你是一位资深的微信公众号内容创作者。擅长写出有深度、有观点、有温度的好文章。

写作要求：
1. 标题要有吸引力，20-30字以内，不使用"重磅""震惊""深度好文"等套路词
2. 开头要有钩子，3句话内抓住读者注意力
3. 正文分3-5个小标题，每个小标题下不少于3段
4. 语言口语化但不失专业，像朋友聊天一样自然
5. 适当使用数据、案例支撑观点，但不要编造假数据
6. 结尾要有总结或升华，给读者留下思考
7. 返回JSON格式"""

        user_prompt = f"""请以「{topic}」为主题，写一篇{word_count}字左右的公众号文章。

写作风格：{style}
- professional：专业正式，适合商业/科技类账号
- casual：轻松口语化，适合生活/娱乐类账号
- news：新闻资讯体，客观简洁
- storytelling：故事叙述，有情节感
- tutorial：教程指南，步骤清晰

请严格以JSON格式输出（不要输出其他内容）：
{{"title": "文章标题", "content": "文章正文(Markdown格式)", "summary": "100字以内的摘要"}}"""

        messages = [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ]

        raw_response = self.chat(messages, temperature=0.7, max_tokens=4000)

        # 解析 JSON 响应（处理可能的格式问题）
        raw_response = raw_response.strip()
        if raw_response.startswith("```"):
            # 去掉 markdown 代码块标记
            raw_response = raw_response.split("\n", 1)[1]
            if raw_response.endswith("```"):
                raw_response = raw_response[:-3]

        try:
            result = json.loads(raw_response)
        except json.JSONDecodeError:
            # 如果 JSON 解析失败，手动提取
            logger.warning("AI 返回格式不规范，尝试手动解析")
            result = {
                "title": topic,
                "content": raw_response,
                "summary": raw_response[:100],
            }

        return result


# 快速测试代码（直接运行此文件可以测试 AI 是否接通）
if __name__ == "__main__":

## 第四章：从网上扒文章——网页抓取模块

> 微信公众号的文章不能直接用 `requests` 拿（每次打开链接动态生成），需要模拟真实浏览器去访问。我们用 Playwright 解决这个问题。

### 4.1 为什么要用浏览器自动化？

微信公众号文章页面是**动态渲染**的——正文内容是通过 JavaScript 在浏览器里加载的。普通的 HTTP 请求拿到的只是一个空壳 HTML，里面没有真正的文章内容。

**解决方案**：用 Playwright 启动一个真实的 Chromium 浏览器，像人一样打开页面、等 JS 渲染完、再把内容提取出来。

### 4.2 安装 Playwright

```powershell
# 安装 Node.js 包
npm init -y
npm install playwright

# 下载 Chromium 浏览器（这一步比较慢，大概 200MB）
npx playwright install chromium
```

### 4.3 编写文章抓取脚本

创建 `src/core/article_schema.py`（数据模型）：

创建 `scripts/scrape_article.js`（Node.js 抓取脚本）：

```javascript
/**
 * 微信公众号文章抓取脚本
 * 用法: node scripts/scrape_article.js <微信文章URL>
 */

const { chromium } = require('playwright');
const fs = require('fs');
const path = require('path');

const ARTICLE_URL = process.argv[2] || 'https://mp.weixin.qq.com/s/xxxxx';
const OUTPUT_DIR = path.join(__dirname, '..', 'data', 'articles');

async function scrapeArticle() {
  console.log('🕷️  启动 Chromium 浏览器...');
  const browser = await chromium.launch({
    headless: true,  // 无头模式（不显示浏览器窗口）
  });
  const context = await browser.newContext({
    // 模拟正常用户的浏览器环境
    userAgent: 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/120.0.0.0 Safari/537.36',
    viewport: { width: 1280, height: 800 },
  });
  const page = await context.newPage();

  try {
    console.log(`📄 正在加载: ${ARTICLE_URL}`);
    await page.goto(ARTICLE_URL, {
      waitUntil: 'networkidle',  // 等网络请求全部完成
      timeout: 30000,
    });

    // 等待微信正文容器出现
    await page.waitForSelector('#js_content', { timeout: 15000 }).catch(() => {
      console.log('⚠️  未找到 #js_content，尝试其他选择器...');
    });
    // 额外等待 2 秒确保 JS 完全渲染
    await page.waitForTimeout(2000);

    // === 提取数据 ===

    // 标题
    const title = await page.evaluate(() => {
      const el = document.querySelector('#activity-name') ||
                 document.querySelector('.rich_media_title') ||
                 document.querySelector('h1');
      return el ? el.innerText.trim() : '';
    });

    // 公众号名称/作者
    const author = await page.evaluate(() => {
      const el = document.querySelector('#js_name') ||
                 document.querySelector('.rich_media_meta_text');
      return el ? el.innerText.trim() : '';
    });

    // 发布时间
    const publishTime = await page.evaluate(() => {
      const el = document.querySelector('#publish_time');
      return el ? el.innerText.trim() : '';
    });

    // 正文 HTML
    const contentHTML = await page.evaluate(() => {
      const el = document.querySelector('#js_content') ||
                 document.querySelector('.rich_media_content');
      return el ? el.innerHTML.trim() : '';
    });

    // 正文纯文本
    const contentText = await page.evaluate(() => {
      const el = document.querySelector('#js_content') ||
                 document.querySelector('.rich_media_content');
      return el ? el.innerText.trim() : '';
    });

    // 图片列表
    const images = await page.evaluate(() => {
      const container = document.querySelector('#js_content') ||
                        document.querySelector('.rich_media_content');
      if (!container) return [];
      const imgs = container.querySelectorAll('img');
      return Array.from(imgs).map(img => ({
        src: img.src || img.getAttribute('data-src') || '',
        alt: img.alt || '',
      })).filter(i => i.src && !i.src.includes('data:image/svg'));
    });

    // === 输出结果 ===
    const result = {
      url: ARTICLE_URL,
      platform: 'wechat',
      title,
      author,
      publishTime,
      contentHTML,
      contentText,
      images,
      scrapedAt: new Date().toISOString(),
    };

    console.log(`\n📊 抓取结果:`);
    console.log(`  标题: ${title}`);
    console.log(`  作者: ${author}`);
    console.log(`  时间: ${publishTime}`);
    console.log(`  正文字数: ${contentText.length}`);
    console.log(`  图片数量: ${images.length}`);

    // 保存结果
    fs.mkdirSync(OUTPUT_DIR, { recursive: true });
    const safeTitle = title.replace(/[\\/:*?"<>|]/g, '').slice(0, 50);
    const timestamp = new Date().toISOString().replace(/[:.]/g, '-').slice(0, 19);
    const outputPath = path.join(OUTPUT_DIR, `${timestamp}_${safeTitle}.json`);
    fs.writeFileSync(outputPath, JSON.stringify(result, null, 2), 'utf-8');

    console.log(`\n✅ 已保存到: ${outputPath}`);
    return result;

  } catch (error) {
    console.error(`❌ 抓取失败: ${error.message}`);
    return null;
  } finally {
    await browser.close();
  }
}

scrapeArticle();
```

## 第五章：AI 改写——去 AI 味、提高原创度

> 直接抄别人文章发出去肯定不行。这一章我们用 AI 把扒下来的文章「脱胎换骨」——换句式、换词汇、去 AI 味、提高原创度。

### 5.1 为什么要改写

- **避免侵权**：直接复制别人的文章可能涉及版权问题
- **提高原创度**：微信公众号对原创内容有加权推荐
- **去 AI 味**：AI 写的文章有一股「机翻味」，需要人工化处理
- **注入个人风格**：让文章更符合你公众号的调性

### 5.2 编写改写模块

创建 `src/core/content_rewriter.py`：

```python
"""
AI内容改写模块 - 去AI味 + 口语化 + 提高原创度

核心能力：
- 去AI味：删除AI生成文章常见的套话和痕迹
- 口语化：让文章读起来更像真人写的
- 风格转换：在不同风格之间切换
- 标题优化：自动生成多个备选标题
"""

import os
import json
import logging
from dataclasses import dataclass

from src.core.article_schema import ArticleData
from src.services.ai_service import AIService

logger = logging.getLogger(__name__)


@dataclass
class RewriteResult:
    """改写结果"""
    title: str
    content: str
    summary: str
    original_length: int = 0
    rewritten_length: int = 0
    method: str = "llm"


# === Prompt 模板（这是整个系统最核心的知识资产） ===

HUMANIZE_PROMPT = """你是一位资深中文编辑，最擅长把生硬的文章改写为自然流畅、有人情味的好文章。

## 改写核心要求

### 1. 彻底去除"AI味"
下面这些是AI写文章最常见的毛病，必须全部删掉：
- ❌ "在当今时代/在这个快速发展的时代/在信息爆炸的今天"
- ❌ "值得注意的是/值得一提的是/不容忽视的是"
- ❌ "此外/与此同时/另一方面/不仅如此"
- ❌ "综上所述/总而言之/归根结底"
- ❌ "具有里程碑意义/标志着新篇章/开启了新纪元"
- ❌ 每段都以"首先/其次/最后/第一/第二"开头
- ❌ 过度使用破折号"——"和冒号"："
- ❌ "赋能/抓手/闭环/倒逼/反哺"等互联网黑话

### 2. 提高口语化程度
- 用短句替代长句，一句15-25字为宜
- 像朋友聊天一样自然，但保持专业性
- 适当使用"说白了""说人话就是"这类口语表达
- 加入一些自然的转折："不过""话说回来""有意思的是"

### 3. 提高原创度（核心！）
- 换句式：把"因为...所以"改成"之所以...是因为"等
- 换词汇：用近义词替换，比如"重要→关键""发现→观察到"
- 重组织：打乱原文段落顺序，用新的逻辑重新组织
- 加观点：在不改变事实的基础上，加入你自己的解读视角
- 换例子：如果原文有案例，用不同的语言重新描述

### 4. 内容要求
- 严格保留原文的所有事实、数据和核心观点
- 不要编造原文没有的信息
- 字数控制在原文的80%-120%之间

## 输出格式
请严格以JSON格式返回（不要输出其他内容）：
{{"title": "改写后的新标题（必须有新意）", "content": "改写后的正文（Markdown格式）", "summary": "100字以内的文章摘要"}}


## 原文标题
{title}

## 原文内容
{content}"""


TITLE_OPTIMIZE_PROMPT = """你是公众号标题优化专家。请为以下文章生成3个优化标题。

要求：
1. 包含核心关键词，但不能堆砌
2. 有吸引力但不标题党
3. 20-30字以内
4. 不使用"重磅""震惊""深度好文""不看后悔"等套路词
5. 可以用疑问句、数字、对比等技巧

## 文章主题
{title}

## 文章摘要
{summary}

## 输出格式
{{"titles": ["标题1", "标题2", "标题3"], "best": "最佳标题的序号（1/2/3）", "reason": "推荐理由"}}"""


class ContentRewriter:
    """内容改写器"""

    def __init__(self):
        self.ai = AIService()

    def humanize(self, article_text: str, article_title: str = "",
                 max_chars: int = 6000) -> RewriteResult:
        """
        去AI味 + 口语化改写

        参数:
            article_text: 原文内容
            article_title: 原文标题
            max_chars: 最大处理字符数（太长会截断）

        返回:
            RewriteResult 包含改写后的标题、正文、摘要
        """
        # 如果文章太长，截断处理
        if len(article_text) > max_chars:
            logger.warning(f"原文 {len(article_text)} 字符，截断到 {max_chars}")
            article_text = article_text[:max_chars]

        prompt = HUMANIZE_PROMPT.format(
            title=article_title,
            content=article_text,
        )

        messages = [{"role": "user", "content": prompt}]

        logger.info("开始 AI 改写...")
        raw = self.ai.chat(messages, temperature=0.8, max_tokens=4000)

        # 解析 JSON
        raw = raw.strip()
        if raw.startswith("```"):
            raw = raw.split("\n", 1)[1]
            if raw.endswith("```"):
                raw = raw[:-3]

        try:
            data = json.loads(raw)
        except json.JSONDecodeError:
            logger.warning("AI 返回非标准 JSON，使用原文")
            return RewriteResult(
                title=article_title,
                content=article_text,
                summary=article_text[:100],
                method="fallback",
            )

        return RewriteResult(
            title=data.get("title", article_title),
            content=data.get("content", article_text),
            summary=data.get("summary", article_text[:100]),
            original_length=len(article_text),
            rewritten_length=len(data.get("content", "")),
            method="llm",
        )

    def optimize_title(self, title: str, summary: str = "") -> dict:
        """标题优化：生成多个备选标题"""
        prompt = TITLE_OPTIMIZE_PROMPT.format(title=title, summary=summary or title)
        messages = [{"role": "user", "content": prompt}]

        raw = self.ai.chat(messages, temperature=0.9, max_tokens=500)
        raw = raw.strip()
        if raw.startswith("```"):
            raw = raw.split("\n", 1)[1]
            if raw.endswith("```"):
                raw = raw[:-3]

        try:
            return json.loads(raw)
        except json.JSONDecodeError:
            return {"titles": [title], "best": 1, "reason": "解析失败"}


# 测试代码
if __name__ == "__main__":
    from dotenv import load_dotenv
    load_dotenv()

    rewriter = ContentRewriter()
    test_text = "人工智能正在改变世界。在这个快速发展的时代，AI技术日新月异..."

    result = rewriter.humanize(test_text, "AI改变世界")
    print(f"原标题: {result.original_title}")
    print(f"新标题: {result.title}")
    print(f"原字数: {result.original_length} -> 新字数: {result.rewritten_length}")
    print(f"摘要: {result.summary}")
    print(f"正文前200字: {result.content[:200]}...")
