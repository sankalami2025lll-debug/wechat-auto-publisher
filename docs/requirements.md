# 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
**微信公众号全自动发布智能体**

### 1.2 项目背景
微信公众号是目前最主流的内容分发平台之一，运营者需要持续输出高质量内容以维持粉丝活跃度。然而，内容创作、排版、定时发布等环节耗时巨大。本项目旨在构建一个基于AI的智能发布系统，实现从内容策划、生成、排版到发布的全流程自动化。

### 1.3 项目目标
- 实现微信公众号文章的AI自动生成
- 支持定时/触发式自动发布
- 支持多内容源（热点追踪、知识库、自定义主题）
- 提供人工审核环节确保内容质量
- 完整的发布记录与数据分析

### 1.4 目标用户
- 微信公众号个人/企业运营者
- 需要批量管理多个公众号的MCN机构
- 希望自动化内容运营的自媒体团队

## 2. 功能需求

### FR-001：AI内容生成
| 属性 | 说明 |
|------|------|
| 优先级 | P0-核心 |
| 描述 | 基于用户设定的主题、风格、关键词，调用大语言模型自动生成公众号文章 |
| 输入 | 主题、关键词、风格要求、字数范围、参考素材 |
| 输出 | 完整文章内容（标题、正文、摘要、封面图建议） |
| 验收标准 | 生成内容语法正确、逻辑通顺、符合公众号格式规范 |

#### FR-001-1：多风格支持
- 支持新闻资讯、深度分析、教程指南、故事叙述等多种写作风格
- 每种风格有对应的Prompt模板

#### FR-001-2：热点追踪
- 接入新闻/热点API，自动获取当日热点话题
- 自动筛选与公众号定位匹配的热点
- 基于热点生成时效性内容

#### FR-001-3：内容优化
- 自动SEO关键词优化
- 标题吸引力评分与多版本生成
- 段落结构优化（小标题、列表、引用等）

### FR-002：自动排版
| 属性 | 说明 |
|------|------|
| 优先级 | P1-重要 |
| 描述 | 将AI生成的Markdown内容转换为微信公众号富文本格式 |
| 验收标准 | 排版美观、图片位置合理、代码块格式正确 |

- Markdown到微信富文本的自动转换
- 支持自定义排版模板
- 封面图自动生成/推荐
- 正文配图自动插入

### FR-003：智能发布
| 属性 | 说明 |
|------|------|
| 优先级 | P0-核心 |
| 描述 | 通过微信公众号API实现文章的自动发布 |
| 验收标准 | 文章成功推送到公众号后台，用户可查看已发布记录 |

#### FR-003-1：定时发布
- 支持设置发布时间表（每日/每周/自定义Cron）
- 支持时区设置

#### FR-003-2：群发与发布
- 支持群发（订阅号每日1次）
- 支持发布（服务号）
- 发布前预览确认

### FR-004：人工审核
| 属性 | 说明 |
|------|------|
| 优先级 | P0-核心 |
| 描述 | 在自动发布前提供人工审核环节 |
| 验收标准 | 审核通过后文章正常发布，驳回后可修改后重新提交 |

- 生成内容需经人工审核确认后发布
- 支持修改标题、内容、封面
- 支持审核驳回并附修改意见
- 审核记录可追溯

### FR-005：多公众号管理
| 属性 | 说明 |
|------|------|
| 优先级 | P2-可选 |
| 描述 | 支持同时管理多个微信公众号 |
| 验收标准 | 可切换不同公众号，发布到指定账号 |

### FR-006：发布记录与分析
| 属性 | 说明 |
|------|------|
| 优先级 | P1-重要 |
| 描述 | 记录所有发布历史，提供统计分析 |
| 验收标准 | 可查看历史发布记录、统计数据 |

- 发布历史列表（时间、标题、状态）
- 发布成功率统计
- AI Token消耗统计
- 内容分类统计

### FR-007：素材库管理
| 属性 | 说明 |
|------|------|
| 优先级 | P2-可选 |
| 描述 | 管理图片、视频等素材资源 |
| 验收标准 | 支持素材上传、分类、检索 |

## 3. 非功能需求

### NFR-001：可靠性
- AI服务不可用时自动切换备用模型
- 发布失败自动重试3次
- 系统异常时保存当前进度，支持断点恢复

### NFR-002：性能
- 单篇文章生成时间 < 60秒
- 排版转换时间 < 5秒
- API调用超时设置 30秒

### NFR-003：安全性
- 所有API密钥加密存储
- 文章内容传输加密
- 日志脱敏处理
- 访问权限控制

### NFR-004：可维护性
- 模块化设计，低耦合高内聚
- 完善的日志系统
- 配置与代码分离
- 完整的单元测试覆盖

### NFR-005：可扩展性
- 支持插件式内容源扩展
- 支持接入多种LLM服务
- 支持自定义排版模板
- 预留多平台发布接口（头条号、知乎等）

## 4. 系统架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户交互层                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │ CLI 命令  │  │ Web 界面 │  │ 定时调度  │               │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘               │
│       └──────────────┼────────────┘                      │
├──────────────────────┼──────────────────────────────────┤
│              智能体核心层 (Agent Core)                    │
│  ┌───────────────────┴───────────────────┐               │
│  │         工作流编排引擎                  │               │
│  │  (内容规划 → 生成 → 排版 → 审核 → 发布)│               │
│  └───────┬───────────────────┬───────────┘               │
│          │                   │                           │
│  ┌───────┴───────┐  ┌───────┴───────┐                   │
│  │  内容生成模块  │  │   发布模块    │                    │
│  └───────┬───────┘  └───────┬───────┘                   │
├──────────┼──────────────────┼──────────────────────────┤
│          │      服务层       │                           │
│  ┌───────┴───────┐  ┌───────┴───────┐                   │
│  │   AI服务      │  │ 微信API服务   │                    │
│  │ (LLM调用)     │  │ (公众号接口)  │                    │
│  └───────────────┘  └───────────────┘                   │
├─────────────────────────────────────────────────────────┤
│                    基础设施层                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │ 配置管理  │  │ 日志系统  │  │ 数据存储  │               │
│  └──────────┘  └──────────┘  └──────────┘               │
└─────────────────────────────────────────────────────────┘
```

### 4.2 核心数据流

```
[用户输入] → [主题分析] → [素材收集] → [AI内容生成]
    → [内容排版] → [人工审核] → [发布] → [记录存储]
```

### 4.3 技术架构选型理由
- **Python**：AI/ML生态最完善，微信SDK支持好
- **LangChain**：标准化LLM调用流程，支持多模型切换
- **Pydantic**：强类型数据校验，确保数据安全
- **loguru**：开箱即用的日志方案，比标准logging更友好

## 5. 接口规格

### 5.1 微信公众号API
- 获取Access Token
- 上传图文素材
- 发布草稿
- 获取发布状态
- 获取素材列表

### 5.2 AI模型API
- 文本生成（Chat Completion）
- 支持OpenAI格式的API（兼容国内模型）
- 支持流式输出

### 5.3 热点API（可选）
- 微博热搜
- 百度热搜
- 知乎热榜

## 6. 数据模型

### 6.1 文章 (Article)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 唯一标识 |
| title | str | 文章标题 |
| content | str | 正文内容(Markdown) |
| summary | str | 摘要 |
| tags | list[str] | 标签 |
| status | Enum | draft/review/approved/published/failed |
| created_at | datetime | 创建时间 |
| published_at | datetime | 发布时间 |
| token_usage | dict | Token消耗记录 |

### 6.2 发布记录 (PublishRecord)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 唯一标识 |
| article_id | UUID | 关联文章 |
| publish_time | datetime | 发布时间 |
| status | Enum | success/failed |
| media_id | str | 微信素材ID |
| error_message | str | 错误信息 |

## 7. 验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|---------|
| AC-001 | 内容生成 | 输入主题后生成完整文章，内容连贯、格式正确 |
| AC-002 | 自动排版 | Markdown正确转换为微信富文本格式 |
| AC-003 | 定时发布 | 设定时间后文章准时发布 |
| AC-004 | 人工审核 | 审核流程完整，支持通过/驳回/修改 |
| AC-005 | 异常处理 | API异常时自动重试，失败后保存现场 |
| AC-006 | 日志记录 | 所有关键操作有日志，可回溯 |
| AC-007 | 密钥安全 | .env不提交，密钥不打印到日志 |
